Dynamic Programming Concept

CLO-2 (Searching): We are searching for the best cut by checking all possible i values from 1 to j. If we skip checking some cuts, we may miss the optimal solution.

CLO-3 (DP Paradigm): This algorithm uses Dynamic Programming because:

  • It breaks the problem into smaller subproblems (optimal substructure)
  • It stores results in a table r[0…n] (memoization/tabulation)
  • It avoids recomputation (shown by red nodes in recursion tree)

CLO-1 (Worst Case): Without DP, recursion explores all 2^(n−1) possibilities — exponential time!
